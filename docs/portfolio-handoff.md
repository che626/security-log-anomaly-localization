# 简历与面试交付

以下内容只使用仓库中已验证的公开实验和真实 ISCC 事实，不把排名 277 包装成奖项，也不把 HDFS 100k 子集说成全量 HDFS。

## 推荐简历条目

**安全日志异常检测与定位｜个人项目｜Python / PyTorch / scikit-learn**

- 将 ISCC 系统日志赛题原型重构为可复现的公开日志基准：实现 HDFS 块级检测、BGL 区间定位、数据清单 SHA256、随机/时间/模板隔离三种切分及 58 项自动化测试。
- 在 104,815 行 HDFS 公开子集（7,940 个块序列）上，比较稀有度规则、TF-IDF 逻辑回归、决策树、Isolation Forest 与 CNN+BiGRU；神经模型在固定随机切分上取得 ROC-AUC 0.9044、PR-AUC 0.6570、F1 0.6383。
- 通过时间与模板隔离评估分析泛化风险：模板隔离下神经模型 ROC-AUC 0.8886 但验证集阈值迁移后 F1 降至 0.4138，据此明确区分排序能力、阈值校准与跨系统泛化限制。

若简历篇幅只有一条，优先使用第二条与第三条合并后的版本；不要写“国三”“获奖”或“全量 HDFS 最优”。

## 30 秒项目介绍

我在 ISCC 数据安全赛中做过系统日志异常定位，最终官方排名 277，没有获奖。赛后我没有把它停留在比赛代码，而是重构成公开可复现项目：对 HDFS 100k 子集把 10 万多行日志按块聚合，和四类传统模型在随机、时间、模板隔离三种切分上对比。模型在随机切分 ROC-AUC 达到 0.9044，但模板隔离 F1 明显下降，所以我把它作为泛化和阈值迁移问题来分析，而不是只展示一个最好看的分数。

## 3 分钟讲述结构

1. **问题与动机（30 秒）**：比赛题既要检测异常，又要定位异常区间和类别；公开数据多数只有二分类标签，因此首先把私有十分类与公开二分类拆开。
2. **数据协议（45 秒）**：HDFS 以 block 为一个序列；BGL 以非重叠窗口和连续异常 span 表示。原始数据不进 Git，清单记录 SHA256、源文件、窗口规则和分区。
3. **模型与基线（45 秒）**：保留原模型的日志规范化、EmbeddingBag、CNN、BiGRU 和边界头；用任务 profile 关闭不适用的类别头。再和稀有度、TF-IDF+逻辑回归、决策树、Isolation Forest 公平比较。
4. **结果与反思（45 秒）**：HDFS 随机切分里神经模型 ROC-AUC 0.9044，但 F1 不是最高；模板隔离 ROC-AUC 仍高而 F1 降低。说明模型排序尚可，验证集选出的阈值不一定可迁移到未见模板。
5. **下一步（15 秒）**：扩展完整 BGL 与固定 Thunderbird 子集，比较零样本、少样本微调和目标域训练，重点报告跨系统误报和阈值稳定性。

## 高频追问与回答要点

| 追问 | 回答要点 |
| --- | --- |
| 为什么不只报随机 F1？ | 随机切分可能让相近模板和相邻日志同时进入训练/测试；时间和模板隔离更接近真实迁移。 |
| 模板隔离 F1 低意味着什么？ | 模型的分数排序仍有 ROC-AUC 0.8886，但验证集阈值在新模板上不稳定；这是需要校准或域适配的证据。 |
| 为什么传统模型有时更好？ | 数据量小、标签模式直接、任务近似线性时，TF-IDF 或树模型更匹配；基线胜出应保留而不是隐藏。 |
| 如何避免数据泄漏？ | 分割在窗口生成前约束源行与分组；词表、频率、阈值、温度均只从训练/验证数据拟合；结果记录绑定清单哈希。 |
| 比赛没有获奖还有什么价值？ | 它提供真实问题与初始模型；真正展示能力的是随后完成的工程化、复现、对照实验、失败分析和边界意识。 |
